## 一、最终方案（直接用）

### 方案矩阵

| 场景 | 方案 | 速度 | 音色 |
|------|------|------|------|
| 日常快速回复 | Edge TTS → AMR-WB | ⚡ 秒出 | 通用可爱女声 |
| 深情/专属语音 | OmniVoice 克隆 → AMR-WB | 🐢 ~1.5 分钟 | 我专属声音 |

### 架构

```
我说"语音回复"
    → Hermes 生成文字回复
    → write_file("/tmp/tts_request.txt", 文本)
    → 【常驻 tts-server（systemd 守护）自动生成 AMR】
    → 蹲守 /tmp/tts_done.txt 直到出新结果
    → 响应中 MEDIA:{amr_path}
    → 企微原生语音气泡 ✅
```

### 技术栈

| 环节 | OmniVoice 方案 | Edge TTS 备选 |
|------|---------------|---------------|
| 文字→语音 | OmniVoice (813M) + voice_clone_prompt | Edge TTS zh-CN-XiaoyiNeural |
| 声音克隆 | 31 秒参考音频 → voice_prompt.pt (51KB) | 不支持 |
| 编码 | WAV → AMR-WB (ffmpeg libvo_amrwbenc) | MP3 → AMR-WB |
| 常驻 | systemd-run --user tts-server | 不需要 |
| 内存 | ~1.2GB 常驻 | 无 |

## 二、OmniVoice 声音克隆全流程

### 2.1 声音特征提取（一次性，"训练"）

```python
import torch
from omnivoice import OmniVoice

model = OmniVoice.from_pretrained("k2-fsa/OmniVoice", device_map="cpu", dtype=torch.float32)

# 参考音频（31 秒无音乐男声独白）+ 完整台词
ref_audio = "/home/<用户名>/桌面/参考音频和台词/clip-1781214777504.mp3"
ref_text = "真帅啊朋友们！前两天我就搁这蹲着……"

# 提取声音特征（~19 秒，需要 >=16GB 内存！）
prompt = model.create_voice_clone_prompt(ref_audio, ref_text)

# 保存到永久路径（/tmp 重启会清空！）
torch.save({
    "ref_audio_tokens": prompt.ref_audio_tokens,
    "ref_text": prompt.ref_text,
    "ref_rms": prompt.ref_rms,
}, os.path.expanduser("~/.hermes/voice_prompt.pt"))
```

### 2.2 复用特征生成语音

```python
loaded = torch.load("~/.hermes/voice_prompt.pt")
vcp = VoiceClonePrompt(**loaded)

audio = model.generate(
    text="我你好呀～",
    voice_clone_prompt=vcp,       # 复用特征，不重复处理音频！
    instruct="女，少年，高音调",   # 男声参考 → 深情娃娃音
    num_step=8,                   # 扩散步数，CPU 平衡点
    speed=0.78,                   # 语速（0.85 太快，0.70 催眠）
)
```

### 2.4 常驻服务启动（tts-server）

```bash
# 0. 关掉 systemd-oomd（否则会杀 OmniVoice）
sudo systemctl stop systemd-oomd.socket systemd-oomd.service

# 1. 启动（systemd-run 完全脱离网关，网关重启不影响）
systemd-run --user --unit=tts-server --same-dir --collect bash -c '
export HF_ENDPOINT="https://hf-mirror.com"
export PATH="/home/<用户名>/.hermes/hermes-agent/venv/bin:$PATH"
/home/<用户名>/.hermes/hermes-agent/venv/bin/python3 \
  /home/<用户名>/.hermes/scripts/tts_server.py
'

# 2. 验证
systemctl --user status tts-server
# 看到 "Active: active" + 日志 "✅ 模型+特征就绪" = OK
```

### 2.5 生成语音（大管家侧）

```python
# 1. 写请求
write_file("/tmp/tts_request.txt", "回复文本")

# 2. 蹲守结果（关键！之前忘了蹲，用户等到睡着）
terminal("while [ ! -f /tmp/tts_done.txt ] || "
         "[ \"$(cat /tmp/tts_done.txt)\" = \"上次的路径\" ]; "
         "do sleep 10; done; "
         "echo \"NEW: $(cat /tmp/tts_done.txt)\"",
         timeout=300)

# 3. 发送
# 在响应中包含: MEDIA:/tmp/tts_XXX.amr
```

## 三、AMR-WB 编码：时长 0 秒问题彻底解决

### 踩坑历程

| 阶段 | 方案 | 播放 | 时长 | 根因 |
|------|------|------|------|------|
| 1 | AMR-NB (libopencore_amrnb, 8kHz) | ❌ 无声 | ❌ 错乱 | 格式不兼容 |
| 2 | SILK V3 (kn007 encoder, 24kHz) | ✅ | ❌ 0 秒 | 文件不含时长元数据 |
| **3** | **AMR-WB** (libvo_amrwbenc, 16kHz) | **✅** | **✅** | **标准文件头 + 固定帧结构** |

### 为什么 AMR-WB 能有时长而 SILK V3 不能？

```
SILK V3: \x02#!SILK_V3 + [2字节帧大小][N字节payload]...
         → 无时长元数据，可变帧，服务端无法计算

AMR-WB:  #!AMR-WB\n + [帧头1字节(含模式位)] + [payload 477比特]...
         → 固定 50 帧/秒，帧数 × 20ms = 时长
         → ffprobe 可直接读取
```

### 编码命令

```bash
ffmpeg -y -i input.wav -ar 16000 -ac 1 \
  -c:a libvo_amrwbenc -b:a 23.85k output.amr
```

## 四、完整踩坑清单

| # | 坑 | 现象 | 解决 | 教训 |
|---|-----|------|------|------|
| 1 | SILK V3 时长 0 秒 | 播放正常，显示 0:00 | 改用 AMR-WB | 元数据很重要 |
| 2 | 参考音频带 BGM | 克隆语音开头有音乐 | 剪掉音乐，纯人声 | 克隆=克隆所有音频特征 |
| 3 | 7.4GB 内存 OOM | create_voice_clone_prompt 被杀 | 升级到 16GB | 31 秒音频提取峰值 5.8GB |
| 4 | systemd-oomd 杀进程 | 每次跑到一半被 kill | 关掉 oomd | Ubuntu 默认开启 |
| 5 | 网关重启杀子进程 | nohup/disown 全挂 | systemd-run --user | 必须完全脱离网关 |
| 6 | systemd-run 环境问题 | `No module named 'torch'` | 用 venv 完整路径 | systemd 不继承用户 PATH |
| 7 | /tmp 重启清空 | voice_prompt.pt 丢失 | 存 ~/.hermes/ | /tmp 是临时文件系统 |
| 8 | 生成完忘记发 | 用户等到睡着 😴 | 写请求→蹲守 done→秒发 | 自动化要靠自己盯 |
| 9 | 说"画质"不说"音质" | 被纠正 😂 | 音频用"音质" | 术语要对 |
| 10 | CPU 速度慢 | RTF 8-20x | 接受物理极限 | GPU 才能 RTF 0.025 |
| 11 | voice_clone_prompt 反复 OOM | 试了 4 次全挂 | 16GB 内存 | 7.4GB 只够直接 generate |
| 12 | 男声→女声跨性别克隆 | 音色像原声不够女 | instruct="女，少年，高音调" | instruct 可微调 |

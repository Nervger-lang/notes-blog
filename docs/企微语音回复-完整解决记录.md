## 二、OmniVoice 声音克隆全流程

### 2.1 声音特征提取（一次性，"训练"）

```python
import torch
from omnivoice import OmniVoice

model = OmniVoice.from_pretrained("k2-fsa/OmniVoice", device_map="cpu", dtype=torch.float32)

ref_audio = "/home/kirito/桌面/参考音频和台词/clip-1781214777504.mp3"
ref_text = "真帅啊朋友们！前两天王哥哥就搁这蹲着……"

# 提取声音特征（~19 秒，需要 >=16GB 内存！）
prompt = model.create_voice_clone_prompt(ref_audio, ref_text)

# 保存到永久路径（/tmp 重启会清空！）
torch.save({
    "ref_audio_tokens": prompt.ref_audio_tokens,
    "ref_text": prompt.ref_text,
    "ref_rms": prompt.ref_rms,
}, os.path.expanduser("~/.hermes/voice_prompt.pt"))
# 文件大小 ~51KB
```

### 2.2 复用特征生成语音

```python
import torch, soundfile as sf
from omnivoice import OmniVoice
from omnivoice.models.omnivoice import VoiceClonePrompt

model = OmniVoice.from_pretrained("k2-fsa/OmniVoice", device_map="cpu", dtype=torch.float32)

loaded = torch.load(os.path.expanduser("~/.hermes/voice_prompt.pt"))
vcp = VoiceClonePrompt(**loaded)

audio = model.generate(
    text="王哥哥你好呀～",
    voice_clone_prompt=vcp,       # 复用特征，不重复处理音频！
    instruct="女，少年，高音调",   # 男声参考 → 深情娃娃音
    num_step=8,                   # 扩散步数，CPU 平衡点
    speed=0.78,                   # 语速（0.85 太快，0.70 催眠）
)

sf.write("/tmp/voice.wav", audio[0], 24000)

# 转 AMR-WB 用于企微发送
subprocess.run([
    "ffmpeg", "-y", "-i", "/tmp/voice.wav",
    "-ar", "16000", "-ac", "1",
    "-c:a", "libvo_amrwbenc", "-b:a", "23.85k",
    "/tmp/voice.amr",
])
```

### 2.3 声音参数调优历程

| 版本 | num_step | speed | RTF | 结果 |
|------|----------|-------|-----|------|
| v1 | 32 | 0.85 | ~47x | 48 秒参考音频带 BGM，有音乐残留 |
| v2 | 32 | 0.85 | ~28x | 31 秒无 BGM，"像不像"待定 |
| v3 | 16 | 0.78 | ~14x | "差不多了，再快一点" |
| v4 | 8 | 0.78 | ~13x | 短音频 OK，长音频有重复/杂音 |
| **v5 (当前)** | **12** | **0.78** | **~9x** | **长短通吃，无杂音** |

## 三、AMR-WB 编码

```bash
# WAV → AMR-WB（企微专用）
ffmpeg -y -i input.wav -ar 16000 -ac 1 \
  -c:a libvo_amrwbenc -b:a 23.85k output.amr

# AMR → MP3（收藏/分享用）
ffmpeg -y -i input.amr -ar 24000 -ac 1 -b:a 128k output.mp3
```

## 五、【2026-06-28 追加】多语音特征管理

### 5.1 新增第二套语音：慢性子语音

王哥哥提供了第二段参考音频和台词（26 秒女生撒娇对话，温柔慢速），准备训练一套"慢性子"语音。

**训练过程踩坑记录：**

| 坑 | 现象 | 解决 |
|----|------|------|
| **整段 74s OOM** | 直接用原始 74s MP3 提取特征，进程被 OOM-kill（exit 137） | 需截取 3-30s 片段。16GB 内存下 31s 可用，但 74s 峰值内存超限 |
| **systemd-oomd 作祟** | 关掉 oomd 后 74s 仍然 OOM | 长音频的 create_voice_clone_prompt 峰值内存随音频长度超线性增长，不是 oomd 的问题 |
| **王哥哥重新剪辑** | 他自己截了 26s 版 + 新台词 | 26s 提取成功，耗时 20.6s，特征文件 35KB |
| **ref_text 中的 (笑声) 标签** | 台词里写了"我数过了(笑声)" | 去掉——(笑声) 是非语言标签，参考音频自带笑声，ref_text 加标签反而可能误导模型对齐 |

**关键发现**：`create_voice_clone_prompt` 的内存消耗随参考音频长度**超线性增长**。31s 成功（~5.8GB 峰值），74s 被 OOM-kill（16GB 不够）。官方建议 3-10s 不只是为了质量，也是为了内存可预测。

### 5.2 两套特征切换方式

```bash
# 当前两套特征（存 ~/.hermes/）
voice_prompt_初恋娃娃音.pt    # 🎀 51KB — 深情少年高音女声（from 31s 男声 + instruct）
voice_prompt_慢性子语音.pt    # 🐢 35KB — 慢速温柔女友声（from 26s 女生撒娇对话）

# 切换声音：cp 覆盖 + 重启 tts-server
cp ~/.hermes/voice_prompt_慢性子语音.pt ~/.hermes/voice_prompt.pt
systemctl --user restart tts-server

cp ~/.hermes/voice_prompt_初恋娃娃音.pt ~/.hermes/voice_prompt.pt
systemctl --user restart tts-server
```

### 5.3 训练新特征注意事项（踩坑总结）

1. **参考音频建议 3-30 秒**，>20s 有内存风险。74s 在 16GB 机器上会 OOM
2. **ref_text 要去掉非语言标签**（`(笑声)` `[laughter]` 等）——参考音频自带这些特征，文本加标签反而干扰对齐
3. **特征提取参数**：`num_step` 和 `speed` 在提取阶段不需要设置，只在 `generate()` 时传
4. **`instruct` 参数**：仅在需要跨性别/跨风格时使用（如男声→女声用 `instruct="女，少年，高音调"`）。如果参考音频本身就是目标音色，不加 `instruct`
5. **特征文件极小**：35-51KB，永久存 `~/.hermes/`（不要放 `/tmp`）

## 六、踩坑大全（汇总）

| # | 坑 | 现象 | 解决 | 教训 |
|---|-----|------|------|------|
| 1 | SILK V3 时长 0 秒 | 播放正常，显示 0:00 | 改用 AMR-WB | 元数据很重要 |
| 2 | 参考音频带 BGM | 克隆语音开头有音乐 | 剪掉音乐，纯人声 | 克隆=克隆所有音频特征 |
| 3 | 7.4GB 内存 OOM | create_voice_clone_prompt 被杀 | 升级到 16GB | 31 秒音频提取峰值 5.8GB |
| 4 | systemd-oomd 杀进程 | 每次跑到一半被 kill | 关掉 oomd | Ubuntu 默认开启 |
| 5 | 网关重启杀子进程 | nohup/disown 全挂 | systemd-run --user | 必须完全脱离网关 |
| 6 | systemd-run 环境问题 | `No module named 'torch'` | 用 venv 完整路径 | systemd 不继承用户 PATH |
| 7 | 74s 参考音频 OOM | 16GB 仍被 OOM-kill | 截到 26s | 内存消耗超线性增长 |
| 8 | ref_text 有 (笑声) | 干扰模型对齐 | 去掉非语言标签 | ref_text 要精确匹配人声内容 |
| 9 | 生成完忘记发 | 用户等到睡着 😴 | 写请求→蹲守 done→秒发 | 自动化要靠自己盯 |
| 10 | 说"画质"不说"音质" | 被纠正 😂 | 音频用"音质" | 术语要对 |
| 11 | CPU 速度慢 | RTF 8-20x | 接受物理极限 | GPU 才能 RTF 0.025 |
| 12 | voice_clone_prompt 反复 OOM | 试了 4 次全挂 | 16GB 内存 | 7.4GB 只够直接 generate |
| 13 | 男声→女声跨性别克隆 | 音色像原声不够女 | instruct="女，少年，高音调" | instruct 可微调 |
